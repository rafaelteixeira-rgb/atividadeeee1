## 📂 Estrutura de Diretórios Planejada

Para garantir a melhor organização do código e dos materiais de apoio, o repositório está estruturado da seguinte forma:

* **`projetos-academicos/`**: Destinado aos projetos, códigos e desafios desenvolvidos em sala de aula e atividades práticas da instituição.
* **`projetos-pessoais/`**: Espaço reservado para projetos independentes, estudos autodidatas e testes de novas tecnologias.
* **`documentacao/`**: Pasta contendo relatórios, slides de apresentação e outros documentos relevantes da trajetória acadêmica.
* **`index.html`**: Página web principal do portfólio, publicada utilizando o **GitHub Pages**.

---

## 🛠️ Tecnologias e Boas Práticas Utilizadas

* **Git**: Utilizado para o controle de versão local do projeto, garantindo o histórico de alterações através de commits descritivos.
* **GitHub**: Plataforma utilizada para hospedagem do código na nuvem e compartilhamento com recrutadores e avaliadores.
* **GitHub Pages**: Serviço de deploy para transformar a página `index.html` em um site estático público.
* **Markdown**: Linguagem de marcação utilizada para a construção e formatação profissional deste arquivo de documentação.
